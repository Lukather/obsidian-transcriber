## Key Decisions

- All tags (`#client`, `#folder`, `#project`, etc.) are passed together to the LLM as a single context block; the LLM synthesizes them into a final path.
- When LLM fails or returns an unparseable response, `isDefaulted = true` and status is `'Defaulted'`; content is NOT cleaned of tags in this case.
- `parseLlmResponse` extracts the first non-empty line, strips backticks/quotes/slashes, and rejects responses over 300 chars.
- `filingModel` setting is now actively used for LLM calls (previously "reserved for future use").
