# 2026-04-14

## Accomplished

- Added dual-provider transcription support with local Ollama and OpenAI-compatible API integration.
- Implemented provider selection in settings with endpoint/API key/model configuration and generation parameters.
- Added provider-aware command behavior (Ollama-only install/remove model actions; generic model selection support).
- Added unchanged-image skip optimization for repeated overwrite runs using persisted file fingerprint + config signature cache.
- Updated batch result notices to report `transcribed`, `skipped`, and `failed` counts.
- Removed provider branding references and standardized wording to OpenAI-compatible naming.
- Updated plugin metadata/support branding to fork maintainer identity.
- Released fork tag `1.0.1` and validated build/lint/typecheck/tests.

## Key Decisions

- Keep provider abstraction (`AiProviderService`) to allow future provider additions without changing orchestration logic.
- Use OpenAI-compatible endpoint semantics (`/models`, `/chat/completions`) instead of provider-specific branding.
- Persist transcription skip cache in plugin data to speed up repeated folder runs while preserving overwrite semantics.

## Domain Model Changes

- `PluginSettings` now includes OpenAI-compatible configuration fields (`openAiBaseUrl`, `openAiApiKey`).
- `PluginSettings` now includes `transcriptionCache` keyed by source file path.
- `TranscriptionResult` now supports optional `skipped` flag for unchanged-file optimization.
